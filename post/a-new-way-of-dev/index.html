A new way of Web development

In this post, I will write about my thoughts on generating site using Hugo as a web developer.

Firstly, I have not used Hugo much but it seems like if you know how to customize it, you can really use it to make all sorts of website.

In terms of the value of the site, it seems like sites generated using hugo will be hard to sell due to the poor usability of the tool. This is because the people that finish developing the website may not be the people that updates it constantly. Eg. Developers can create the wordpress platform, but the people that are using it the most are people that are in charge of content such as editors, marketing team etc. So usability is a huge hurdle for Hugo to overcome. I think an improvement to Hugo would maybe be an app with an improved user interface that allows user to update post or create the site without ever touching the command line. That would then gives us developers value to use this to build our sites.

The ease of updating the site is also an issue. For sites like wordpress, you can easily update the page by going to the admin menu, edit it and click the save button. But for Hugo site, you need to edit the page, save it, generate the hugo site, overwrite the server hugo site with the newly generated files before it is done. This creates alot of trouble especially if you update your site several times a day.

So who are Hugo good for now? I think for companies that needs a fast loading webpage and do not mind hiring a tech savy editor. Why do I say that, because the person still really needs to be comfortable with using git, or being able to access the server using ftp and copying the public folder generated and overwriting it to update content. People that are not tech savvy might run into problems and when they cant fix the problems, site might go down or they might even destroy the source files. So definitely place Hugo sites in the hands of people that knows what they are doing.

Also if you need to generate a site that rarely needs update, then truthfully you can just hire a designer to build a static page using html css, etc. You really do not need Hugo if all you need is a static webpage that does not need updating.

Issues

  • Lack of user interface may make it hard for customer to update the site in future
